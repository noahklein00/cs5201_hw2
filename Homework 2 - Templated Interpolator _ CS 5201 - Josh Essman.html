Homework 2 - Templated Interpolator

Purpose

This assignment is designed to refamiliarize you with writing OO code in C++, and in particular defining and implementing good class design. Some of the concepts addressed in this assignment may be review for you and some may not. But they are very fundamental to object-oriented design and implementation of numerical simulations. Thus, you are going to be using:

  • templated classes
  • templated functions
  • operator overloading
  • the STL, in particular the vector and tuple classes, as well as the algorithms library
  • the concept of implementing operations as classes
  • C++ to “wrap” a data type

Background

Throughout the semester we will be focusing on the application of object-oriented design principles to numerical simulations of complex physical systems. In this introductory assignment, you will be implementing a basic simulation method - the piecewise linear model.

Most simulations in academia and in industry attempt to replicate the behavior of systems governed by equations which make conventional (i.e., analytical) techniques ineffective. These equations might be not fully understood or might be computationally intractable. One solution to this problem is to simply collect data from the physical system and then interpolate to determine the values of the dependent variables for a given set of inputs. This approach is not generally used for high-dimensional models (those with a large number of independent variables), as the runtime complexity is exponential with respect to the dimensionality of the system. This piecewise linear approach is attractive because it does not require any understanding of the underlying system, but in general only performs well when the system behaves approximately linearly between a given pair of samples.

Program Specification

You will be responsible for implementing a templated linear interpolator class, which should use C++ templates to support interpolation for data of any arbitrary type that satisfies your preconditions.
This interpolator will deviate slightly from the linear interpolation you learned in CS 3200, as it must handle a function of one independent and two dependent variables, i.e.:

F:R→R×RF:R→R×R

Note that this is different from bilinear interpolation.

You will be given a three-column data file corresponding to the start sequence for a conventional fission nuclear reactor, whose independent variable will be strictly increasing (though this should still be checked - consider the STL for this). The first column will always be the independent variable. For the nuclear reactor data, your independent variable will be time (s), and your two dependent variables will be reactivity, ρρ, which is dimensionless, and reactor power output (watts).

You will be responsible for implementing the following operations for your templated interpolator class:

  • parameterized constructor accepting a std::vector of points (which are of type std::tuple)
  • parameterized constructor accepting the number of points in the dataset, which should default to 20
  • copy constructor and copy assignment (=)
  • subscript operator ([]) for accessing or modifying the 3-tuple with the closest independent variable value less than or equal to the index parameter - note that you should support access-only functionality
  • function evaluation operator () for interpolating given a valid value of the independent variable, returning the values of the dependent variables
  • extraction operator (>>) for reading in a given number of points from a std::istream - points read in should overwrite any existing data
  • insertion operator (<<) for pretty-printing every point in the dataset to a std::ostream such that 128.1: -55.7, 12.6 indicates that the independent variable 128.1 corresponds to dependent variable values -55.7 and 12.6, respectively. Newlines should separate each data point.
  • an overload of the bitwise complement operator (~) to return the range of the independent variable for a given interpolator object (a scalar representing the span)
  • less-than/greater-than/logical equality operators (</>/==/!=), s.t. for interpolators a and b, a < b is true if b's range of the independent variable is greater than a's, and a == b if the range of the independent variables for a and b are equal

You should not use std::upper_bound, std::lower_bound, std::equal_range, or std::binary_search in this assignment.

As always, you are expected to follow the assignment guidelines. Since this is the first programming assignment, please read through the above guidelines completely.
